Sobre o Sistema

O Festou é um sistema web de gestão para empresas de eventos e festas. Centraliza controle de estoque, vendas, locações, financeiro e recursos humanos em um único ambiente acessado pelo navegador.

Tecnologia: Aplicação web (SPA) com banco de dados Supabase (PostgreSQL). Todos os dados são salvos em tempo real na nuvem, sem necessidade de instalação local.

Principais recursos

  • Controle de produtos, saldos de estoque e armazéns
  • Vendas, locações e emissão de cupom fiscal
  • Gestão de clientes, fornecedores e vendedores
  • Agenda de eventos com controle de devoluções
  • Financeiro completo: contas bancárias, extrato, despesas e fechamento de caixa
  • Módulo de RH: cadastro de funcionários, ponto eletrônico, banco de horas e holerites
  • Sistema de permissões por rotina, por operador e por filial

Acesso e Login

Como entrar no sistema

  1. Abra o sistema no navegador
  2. Selecione a Filial na qual vai trabalhar
  3. Informe seu Login e Senha
  4. Clique em Entrar
⚠️ Acesso por filial: Cada operador é vinculado a uma filial específica. Superadmins (sem filial vinculada) podem acessar qualquer filial.

Sessão

A sessão fica ativa enquanto o navegador estiver aberto. Ao fechar ou clicar em Sair, será necessário fazer login novamente. Os dados de sessão não persistem entre dispositivos.

Navegação

O sistema usa um menu lateral (sidebar) com todas as rotinas disponíveis. As rotinas visíveis dependem das permissões do operador e dos módulos habilitados para a filial.

Menu Lateral

As rotinas são agrupadas por categoria: Estoque, Comercial, PDV, Financeiro, Recursos Humanos e Administração. Clique em qualquer item para abrir a rotina na área principal.

Dashboard (Home)

A tela inicial exibe cards de acesso rápido para todas as rotinas que o operador tem permissão de usar. Os cards espelham automaticamente a visibilidade do menu lateral — se um módulo não está habilitado, o card não aparece.

Tema

O sistema suporta modo claro e modo escuro. Alterne pelo botão de tema no topo do sidebar. A preferência é salva no navegador.

Sistema de Permissões

O acesso às rotinas é controlado por dois níveis:

Nível 1 — Módulos da Filial

Cada filial tem um conjunto de módulos habilitados (configurado em Filiais → Rotinas). Mesmo que um operador tenha permissão individual, se o módulo não estiver ativo na filial, a rotina não aparece.

PDV — Ponto de Venda

🛒
Ponto de Venda
Vendas, locações, crediário e emissão de cupom
⚠️ O PDV está disponível apenas para operadores da Filial 02.

Tipos de Operação

TipoDescrição
VendaVenda normal. Baixa estoque, gera cupom, registra pagamentos.
LocaçãoAluguel de itens com data de saída e data de devolução prevista. O estoque é debitado na saída e estornado na devolução. Status: pendente → devolvida.

Fluxo de Venda

  1. Adicione produtos ao carrinho (busca por código ou descrição)
  2. Selecione o cliente (opcional — obrigatório para desconto)
  3. Selecione o vendedor (obrigatório)
  4. Aplique descontos se necessário
  5. Selecione a(s) forma(s) de pagamento
  6. Finalize e imprima o cupom

Regras do PDV

  • Máximo 3 formas de pagamento por venda
  • Se houver um único pagamento cobrindo o total e o operador selecionar uma nova forma, o pagamento é substituído (não duplicado)
  • Descontos por item e em lote só são habilitados para clientes do tipo Decorador
  • A seleção de vendedor é obrigatória para finalizar
  • Carrinho vazio ou total ≤ 0 impede a finalização
  • Pagamento com valor 0 não é processado

Formas de Pagamento

  • Dinheiro — com cálculo de troco automático
  • Pix
  • Crédito
  • Débito
  • Crediário — pagamento a prazo, gera conta a receber com data de vencimento

Crediário

Vendas no crediário ficam com status_crediario = 'pendente' até serem recebidas. O recebimento é feito na rotina Contas a Receber. A data de vencimento é definida no momento da venda.

Histórico de Vendas

📊
Histórico de Vendas
Consulta, estorno e reimpressão de vendas realizadas

Funcionalidades

  • Listagem com filtro de período e status (finalizadas / canceladas)
  • Cards de resumo: locações pendentes, crediários pendentes, total de vendas, receita, descontos, ticket médio
  • Modal de detalhe com itens, pagamentos e totais
  • Reimpressão de cupom
  • Estorno de venda
  • Registro de devolução de locação

Regras de Estorno

  • Apenas vendas com status finalizada podem ser estornadas
  • O estorno restaura o saldo do estoque de todos os itens
  • Cria movimentações financeiras negativas para cada forma de pagamento
  • Exige confirmação do admin ou usuário com acesso_pdv

Devolução de Locação

  • Disponível para locações com status pendente
  • Pode incluir uma taxa de atraso com conta bancária de destino
  • Atualiza o status para devolvida
  • Cria evento na agenda documentando a devolução

Contas a Receber

📒
Crediários e Contas a Receber
Gerencie e baixe pagamentos a prazo

Funcionalidades

  • Listagem de vendas crediário com filtro de período e status
  • Badge de vencido (pulsante vermelho) para crediários em atraso
  • Recebimento com seleção de conta bancária de entrada
  • Ao receber: cria movimentação financeira + atualiza status + cria evento na agenda

Regras

  • O filtro padrão mostra o mês atual (1º ao último dia)
  • Apenas crediários com status pendente exibem o botão "Receber"
  • Crediários recebidos mostram badge verde com ✓
  • Crediários vencidos (data de vencimento anterior à data atual e ainda pendentes) mostram badge vermelho pulsante

Financeiro

💳
Contas Bancárias e Extrato
Visão geral de saldos e movimentações financeiras

Abas do módulo

Contas

  • Grid com todas as contas ativas da filial
  • Saldo em tempo real (verde = positivo, vermelho = negativo)
  • Tipos de conta: Pix, Débito, Crédito, Dinheiro (Caixa Físico)

Nova Conta

  • Crie uma conta bancária com nome, tipo e saldo inicial
  • O campo "nome" é obrigatório
  • O saldo inicial é registrado como primeira movimentação

Extrato

  • Lista de movimentações filtradas por conta
  • Ordenadas da mais recente para a mais antiga (últimas 100 movimentações)
  • Entradas (verde) e saídas (vermelho)
  • Vinculação com venda (venda_pk) quando aplicável
Atualização automática de saldo: Um trigger no banco de dados atualiza o saldo da conta automaticamente a cada movimentação inserida. O sistema nunca precisa calcular o saldo manualmente.

Despesas

📉
Contas a Pagar e Despesas
Controle de saídas financeiras e pagamentos a vencer

Funcionalidades

  • Criar despesas com descrição, fornecedor (opcional), categoria (texto livre), valor e vencimento
  • Marcar como já pago no momento da criação (requer seleção de conta)
  • Marcar despesas pendentes como pagas (seleciona conta de débito)
  • Badge de vencido para despesas em atraso
  • Filtro padrão: mês atual

Regras

  • Campos obrigatórios: descrição, valor e vencimento
  • Ao pagar, cria uma movimentação de saída na conta selecionada
  • Despesas pagas ficam bloqueadas para edição de status (não é possível reverter para pendente pela interface)
  • Despesas vencidas e ainda pendentes recebem badge vermelho

Fechamento de Ponto

🧾
Banco de Horas e Fechamento do Período
Apuração do período, cálculo de salário e bloqueio para pagamento

Conceito

O fechamento é feito por quinzena (1ª: 01 a 15 / 2ª: 16 ao fim do mês). Cada fechamento calcula as horas trabalhadas, compara com as horas previstas, gera o saldo do banco de horas e calcula o valor líquido a pagar.

Fluxo Completo (com Espelho)

  1. Admin abre o fechamento do funcionário no período desejado
  2. Confere os dias, batidas e justificativas na tabela
  3. Lança descontos e define horas extras a pagar, se houver
  4. Clica em "Enviar Espelho para Aprovação"
  5. O funcionário acessa Espelho de Ponto e aprova ou rejeita
  6. Se rejeitado, o admin revisa e reenvia o espelho
  7. Após aprovação, o botão "Salvar e Bloquear Período" é habilitado
  8. Admin finaliza — o período fica bloqueado e o saldo do banco vai para o próximo período
🔒 O período só pode ser finalizado após o funcionário aprovar o espelho de ponto. Tentar finalizar sem aprovação exibe um erro.

Cálculos do Período

CampoCálculo
Horas PrevistasCarga horária diária × dias úteis no período (fins de semana = 0)
Horas Trabalhadas(Última batida − Primeira batida) − Intervalo configurado, por dia
Saldo do PeríodoTrabalhado − Previsto (positivo = hora extra, negativo = falta)
Saldo AcumuladoSaldo Anterior + Saldo do Período
Salário Proporcional(Horas pagas / Horas do período) × Salário base
Horas Extras Pagas ($)Horas extras × Valor/hora × 1,6 (multiplicador 60%)
Total LíquidoSalário Proporcional + Horas Extras Pagas − Descontos

Dias com Justificativa

  • Dias com justificativa (atestado médico, abono, folga, feriado) têm o previsto zerado — não geram débito de horas
  • Para mensalistas, um dia justificado conta como se o funcionário tivesse trabalhado (paga a carga diária completa)
  • Para diaristas, um dia justificado conta como uma diária paga

Tipos de Justificativa

  • Atestado Médico
  • Abono
  • Folga
  • Feriado
  • Outros

Cálculo para Diaristas

  • Não há horas previstas — diaristas não acumulam banco de horas por falta
  • Pagamento = Número de dias pagos × Valor da diária
  • Dias pagos = dias com batidas registradas + dias justificados

Reabrir Período

Admins podem reabrir um período bloqueado. O saldo acumulado do próximo período não é recalculado automaticamente — o admin deve reabrir e refazer o fechamento subsequente, se necessário.

Espelho de Ponto

🪞
Espelho de Ponto — Visão do Funcionário
Revisão e aprovação do fechamento de ponto pelo próprio colaborador

Acesso

Disponível para operadores com a permissão acesso_espelho_ponto ativada. O sistema vincula automaticamente o operador ao funcionário pela matrícula (campo matricula do operador deve ser igual à matrícula do funcionário).

Funcionalidades

  • Exibe espelhos pendentes de aprovação em destaque amarelo no topo
  • Exibe histórico de espelhos já aprovados ou rejeitados abaixo
  • O funcionário pode expandir cada espelho para ver o detalhamento diário das batidas
  • Botões Aprovar e Rejeitar para cada espelho pendente

Aprovação

O funcionário confirma que concorda com os dados do espelho. Após aprovação, o admin recebe sinal verde e pode bloquear o período no módulo de Fechamento de Ponto.

Rejeição

O funcionário pode rejeitar informando o motivo por texto. O motivo fica visível para o admin no módulo de Fechamento de Ponto. O admin deve corrigir o ponto (batidas, justificativas) e reenviar o espelho.

⚠️ Após a rejeição, o espelho_status volta para "rejeitado". O admin precisa clicar em "Enviar Espelho para Aprovação" novamente para que o funcionário possa ver a versão corrigida.

Meus Holerites

📄
Contracheques do Funcionário
Consulta de holerites de períodos fechados

Acesso

Disponível para qualquer operador com o campo matrícula preenchido. O sistema localiza o funcionário pela matrícula e exibe apenas os holerites vinculados a ele.

Conteúdo do Holerite

  • Identificação: nome, matrícula, período (quinzena, mês, ano)
  • Proventos: salário base, horas extras pagas
  • Descontos detalhados (da tabela descontos_fechamento)
  • Total líquido a receber
  • Banco de horas: saldo anterior, saldo do período, horas extras pagas, saldo final

Regras

  • Somente fechamentos com status bloqueado = true aparecem como holerites
  • Ordenados do mais recente para o mais antigo
  • Apenas leitura — funcionário não pode modificar nenhum dado
  • Formato pronto para impressão

Configuração de Permissões

Na tela de cadastro do operador, cada rotina pode ser habilitada individualmente. Operadores do tipo Admin têm todas as permissões automaticamente, sem necessidade de configuração individual.

Notificações do Sistema

Todas as ações de sucesso ou erro são exibidas por um toast (mensagem temporária) no canto da tela — sem abrir janelas externas do navegador. Confirmações de ações importantes (bloquear período, rejeitar espelho, reabrir período) aparecem em modais internos ao sistema.

✅ Sem pop-ups do browser: O sistema usa exclusivamente modais internos para confirmações, sem usar as caixas de diálogo nativas do Google Chrome ou outros navegadores.
